        Matching Algorithm (V2)
        • 1. Data Sanitization:
          Before matching, the system automatically cleans the data. Account Codes or Names that are blank, a hyphen (`-`), or zero (`0`) are treated as empty to prevent false positives and ensure matches are based on meaningful data.
        • 2. Adaptive Core Logic:
          The algorithm first detects if your trial balances use Account Codes. It then adapts its strategy for the highest accuracy:
          • With Account Codes: A comprehensive 7-pass process is used, prioritizing code matches first, then falling back to name analysis for any remaining items.
          • Names Only: A streamlined 4-pass process is used, focusing exclusively on exact and similar name matching.
        • 3. Detailed Passes (for TBs with Account Codes):
          The system prioritizes matches in the following order of confidence:
          • Pass 1: Exact Match: Identifies pairs with a perfect match on both their Account Code and Account Name.
          • Pass 2: Code-Primary Matching: For accounts with matching codes but different names, the system intelligently categorizes the pair:
            • Name Similarity: Uses a fuzzy-matching algorithm for minor name variations (e.g., "Trade Receivable" vs. "Accounts Receivable").
            • Name Missing: Creates a valid match if one or both names are blank, assigning the status `Code Match (Name Missing !!)` to flag the data quality issue.
            • Name Mismatch: Flags pairs with matching codes but completely different names for mandatory auditor review.
          • Pass 3: Name-Primary Matching: If codes do not match or are not used, the system seeks matches based on the account name:
            • Exact Name Match (Code Missing/Mismatched): Creates a valid match if names are identical but one/both codes are blank (`Name Match (Code Missing !!)`) or if the codes are present but different (`Code Mismatch Name Match`).
            • Similar Name Match (Code Mismatched): As a final step, the system uses fuzzy matching to find pairs with similar names, even if their codes are different, flagging them as `Code Mismatch Name Similarity (XX%)` for careful review.
        • 4. Final Categorization:
          Any account that remains unpaired after these passes is definitively classified as either a `New Account` or `Prior Period Only`.
        • 5. Analysis & Assertions:
          For every matched pair, the system calculates the variance and automatically identifies the most relevant audit assertion (e.g., Accuracy, Completeness, Classification) to guide and streamline your review process.
